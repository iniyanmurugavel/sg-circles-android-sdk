# Circles Roaming Brownfield SDK & Host App

This project demonstrates an enterprise-grade standard for a **Shared React Native Brownfield SDK** embedded into Android and iOS host apps.

> **For build, integration, and publishing guides** — see [`RELEASE_PLAN.md`](./RELEASE_PLAN.md)  
> **For architectural decisions and standards** — see [`ARCHITECTURE_STANDARD.md`](./ARCHITECTURE_STANDARD.md)

## Project Structure

This workspace has two essential components:

1. **`circles-sg-rn-expo/`** (🔥 **The React Native SDK Source**)
   The shared React Native / Expo source code. This is where 100% of the cross-platform logic and UI is developed. When built, it generates the native AAR/XCFramework libraries.

2. **`sg-circles-android-host/`** (🔥 **The Native Android Host App**)
   A fully native Android app that consumes the generated SDK from `circles-sg-rn-expo`. This is the integration proof point.

> **Note:** The SDK distribution registries (`sg-circles-android-sdk`, `sg-circles-ios-sdk`) have been removed from this local workspace. In production, artifacts are published to remote Maven (JitPack / GitHub Packages) or a Swift Package registry. See `RELEASE_PLAN.md` for the full publishing guide.

## Setup & Running the Project

### 1. The Expo React Native App (SDK Source)
We treat the React Native project as the **source of truth** for our Shared SDK. The actual code is in `circles-sg-rn-expo`.
- Go to `circles-sg-rn-expo` and run `npm install`.
- For fast iteration, run `npm run dev` to use Expo Go.

### 2. Generating AAR Builds (Debug & Release)
To build the native Android artifact (`.aar`) containing your React Native code:
```bash
cd circles-sg-rn-expo

# Generate Debug AAR Build (For testing locally in host app with Metro running)
npm run build:android:debug

# Generate Release AAR Build (For production / final Maven publishing)
npm run build:android:release
```

After generation, the `circles-travel-pass-sdk-release.aar` or `circles-travel-pass-sdk-debug.aar` will be created inside `circles-sg-rn-expo/android/circles-travel-pass-sdk/build/outputs/aar/`. 

To locally publish these so the host app can resolve them:
```bash
# From within circles-sg-rn-expo/android:
./gradlew :circles-travel-pass-sdk:publishBrownfieldReleasePublicationToMavenLocal
```

### 3. The Host Android App (`sg-circles-android-host`)
The `sg-circles-android-host` is standard Android project applying the AAR.
- It is Edge-to-Edge configured for **Android 15+**.
- It uses a native Android `Toolbar` built natively to act as a proper native shell.
- It embeds the React Native UI inside `ReactNativeFragment.kt`.

To run the host app:
1. Open `sg-circles-android-host` in Android Studio.
2. Build and run the `app` module on a device or emulator.

## Do's and Don'ts for the Brownfield SDK Architecture

### Do ✅
- **DO define strict contracts:** Use strongly-typed Launch states and Completion handlers (e.g. `circles.roaming.launch` messages).
- **DO delegate Edge-to-Edge to Native:** Ensure the native shell (e.g., `CirclesRoamingContainerActivity`) handles `WindowInsetsCompat` and renders the Toolbar, while the RN UI fills the available content container.
- **DO verify local builds before publishing:** Always test the debug AAR before making a release request.
- **DO let the native host manage its own lifecycles:** Never have React Native assume it owns `finish()` or `System.exit()`. Call completion callbacks directly to native instead.

### Don't ❌
- **DON'T bundle the entire app in the SDK:** This is for shared *features* (like a Travel Pass popup/flow). 
- **DON'T try to manage Native Navigations inside RN:** Use React Navigation *within* the RN module, but exit completely gracefully when done and let Android `Activity` or Jetpack Navigation handle the rest.
- **DON'T commit generated native folders unless required:** The `circles-sg-rn-expo/android` code is mostly autogenerated. The real logic is mapped in TS.
- **DON'T ignore Edge-to-Edge inset calculations:** If the RN screen overlaps with System Bars (Android 15), make sure the Host sets its view boundaries accurately.
