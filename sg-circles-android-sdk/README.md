# sg-circles-android-sdk

JitPack distribution repo for the **Circles Travel Pass SDK** — a pre-built Android AAR generated by the Expo brownfield React Native module in `circles-sg-rn-expo`.

## Why a Flat Structure?

No nested submodule folder needed. The AAR and its `maven-publish` Gradle config live directly at the root. JitPack just runs `./gradlew build` on the root project.

```
sg-circles-android-sdk/
├── build.gradle                        ← maven-publish config + POM declaration
├── settings.gradle                     ← project name only
├── circles-travel-pass-sdk-1.0.0.aar  ← pre-built AAR (generated by expo-brownfield)
├── gradle/                             ← Gradle wrapper
├── gradlew
└── README.md
```

## Artifact Coordinates

```kotlin
implementation("com.github.iniyanmurugavel:sg-circles-android-sdk:v1.0.0")
```

## How to Consume

**`settings.gradle.kts`** in your host app:
```kotlin
dependencyResolutionManagement {
  repositories {
    google()
    mavenCentral()
    maven { url = uri("https://jitpack.io") }
  }
}
```

**`app/build.gradle.kts`**:
```kotlin
dependencies {
  implementation("com.github.iniyanmurugavel:sg-circles-android-sdk:v1.0.0")
}
```

## Updating the AAR

1. Generate new AAR from `circles-sg-rn-expo`:
   ```bash
   npm run build:android:release
   ```
2. Replace `circles-travel-pass-sdk-X.X.X.aar` at the root.
3. Update version in `build.gradle`.
4. Commit, tag new version (`v1.1.0`), push.
5. Host app: `implementation("com.github.iniyanmurugavel:sg-circles-android-sdk:v1.1.0")`
